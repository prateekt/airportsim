Integration Problems ATC/RealWorld/GUI

 

This part of integration involved Eric Quillen and me. We were combining Ajay Prasad and my already integrated back-end with Eric's GUI.

 

There were only a couple of integration issues, because at the time of integration, the 105 team hardly had anything at all for the actual simulation display.  Most of it was me dictating to Eric what we were already prepared to give them, and him making sure his side received it correctly, so that once they have an animation and risk lists, they will have all the data they need to display it.

 

The only actual bug that we found that was due to the integration was that once the GUI received the lists of data from my WorldState object, and had displayed it, he did a list.clear().  The next time he called for an update, he got an empty list. Because i hadn't made a deep copy of my list, we were using shared data and he was clearing the list that i held in my RealWorld Class.  We fixed this by taking out the list.clear() and i made him promise he wouldn't mess with my data again.  But a better solution would be to figure out how to do a deep copy and do that, like we're supposed to.

 

There was also an extra function that Ajay's ATCAssistantAgent called in the GUI that involved functionality that we had since cut for v1.  Naturally, the GUI no longer had this function, but we still had it in our end so it was a bug.

 

Finally, we forgot to make the real GUI implement our Gui interface, something we had set up so we could use a mock gui before integration.  This obviously caused some problems before we figured out what we had done.
