Additional Explanations                                                                                  Jeff Magers

 

RealWorld

The RealWorld was not implemented as an agent, so I did not do a design document the same way I do a design doc for an agent.  This is where I explain the RealWorld and a couple of other loose ends.

 

RealWorld is very simple, but pretty powerful.  Upon creation, it reads in data from the given scenario file.  It stores this data in 3 data structures:

 

List<Plane> planes

Map<Position, Runway> runways

Map<Position, Taxiway> taxiways

 

There is also an update function that can be called from the GUI that returns a WorldState object.  (AWorldState object, as noted in KnoledgeBaseAgent simply holds these 3 previously mentioned data structures as well as a Weather enum).

 

Before it creates a new WorldState object, it calculates the location of each plane.  It does this by simply telling each plane how much time has elapsed since the last update.  The implementation is there and functioning to do this in real time, but because of the nature of the current GUI, we are statically setting the length of each time slice to a set number of milliseconds.

 

The planes then calculate (in their own methods) where they are supposed to be based on the time elapsed. If they were supposed to turn at an intersection, they take this into account and modify their direction and location accordingly.

 

Also, every time an update is called, the RealWorld sends a WorldState object to the KnowledgeBaseAgent so it can calculate risks.

 

 

Risks/Mitigations/Actions

These classes are general top-level risks/mitigations/actions.  There are more specific classes, such as CollisionRisk which extend the appropriate class.  Once the KBAgent identifies and creates a specific risk, that risk creates its own list of Specific Mitigations, each of which creates its own list of specific actions.  So, just by the KB creating a single risk, the entire Risk/Mitigation/Action structure is created.

 

 

KB/ATC Interaction

The KnowledgeBase is able to find multiple risks at a time, but currently the ATC cannot handle getting several risks at once, so we will fix this for v2.
